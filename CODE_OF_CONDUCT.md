# Code of Conduct

## Our pledge

We want Network Device Investigator to be a welcoming, useful, and safe project for people interested in defensive network-device investigation.

Contributors and maintainers are expected to participate respectfully and constructively.

## Expected behaviour

Examples of positive behaviour include:

- Being respectful and professional
- Welcoming new contributors
- Giving constructive feedback
- Explaining uncertainty clearly
- Keeping the project defensive and safe-by-design
- Respecting privacy and confidentiality

## Unacceptable behaviour

Examples of unacceptable behaviour include:

- Harassment, insults, or personal attacks
- Publishing private network details, credentials, customer data, or secrets
- Encouraging unauthorised access
- Contributing brute-force, exploitation, persistence, malware, or credential-guessing workflows
- Deliberately misleading device-identification guidance
- Spam or unrelated self-promotion

## Safety-first project scope

This project is defensive-only.

It exists to help identify, assess, and safely remediate unknown network devices in environments the user owns or is authorised to investigate.

It should not become an offensive exploitation toolkit.

## Enforcement

Maintainers may remove, edit, or reject comments, issues, pull requests, or other contributions that do not align with this code of conduct or the defensive scope of the project.

Repeated or serious violations may result in being blocked from participating in the project.

## Reporting

If you see behaviour that violates this code of conduct, open an issue or contact the maintainer through GitHub.

Do not include sensitive information in public reports.
